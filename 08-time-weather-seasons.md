# 설계 08 — 시간·날씨·계절 시스템

> **전제 조건**: 07 단계 완료 상태.
> 게임 내 시간(GameTime), 맵 렌더링, 파티클 시스템 기반이 구현되어 있다고 가정한다.

---

## 1. 이번 단계 목표

1. 시간대별 화면 밝기 변화 (낮/밤 사이클)
2. 4계절 시스템 (봄·여름·가을·겨울, 각 24일)
3. 계절별 날씨 이벤트 (맑음·비·눈·안개·낙엽 등)
4. 날씨에 따른 게임플레이 영향
5. 시작 조건: 봄 1일 09:00

---

## 2. 시간 구조

### 2-1. 기본 단위 (기존 GameTime 기준)

```
현실 1초 = 게임 내 48초
현실 30분 = 게임 내 1일

게임 내 1시간 = 현실 75초 (1분 15초)
게임 내 1일   = 현실 30분
```

### 2-2. 계절 구조

```
봄  (Spring): 1일 ~ 24일
여름 (Summer): 25일 ~ 48일
가을 (Autumn): 49일 ~ 72일
겨울 (Winter): 73일 ~ 96일

1년 = 96일 = 현실 48시간 (2일)
```

### 2-3. 시작 조건

- 게임 최초 시작: **봄 1일 09:00**
- Firebase `/rooms/{seed}/startTime` 에 게임 시작 현실 타임스탬프 기록
- 모든 클라이언트는 이 값으로부터 현재 게임 내 시간을 계산
  ```
  elapsed_real_sec = now - startTime
  game_minutes = elapsed_real_sec * 48
  game_day = floor(game_minutes / 1440) + 1   // 1440 = 하루 분 수
  game_hour = (game_minutes % 1440) / 60
  ```
  여기에 시작 오프셋(09:00 = 540 game_minutes) 을 더함.

## 3. 시간대별 밝기 (낮/밤 사이클)

화면 전체에 어두운 오버레이를 씌워 밝기를 조절.
오버레이는 Phaser Camera의 `setPostPipeline` 또는
카메라 고정 반투명 Graphics 레이어로 구현.

### 3-1. 시간대 구분

| 시간대 | 게임 내 시각 | 오버레이 색상 | 투명도 (alpha) |
|--------|-------------|---------------|----------------|
| 새벽 전 (심야) | 00:00 ~ 04:00 | 남색 `#000033` | 0.75 |
| 여명 | 04:00 ~ 06:00 | 남색 → 주황 | 0.75 → 0.0 (선형 보간) |
| 아침 | 06:00 ~ 08:00 | 투명 | 0.0 |
| 낮 | 08:00 ~ 17:00 | 투명 | 0.0 |
| 노을 | 17:00 ~ 19:00 | 투명 → 주황 `#331100` | 0.0 → 0.3 (선형 보간) |
| 저녁 | 19:00 ~ 21:00 | 주황 → 남색 | 0.3 → 0.65 (선형 보간) |
| 밤 | 21:00 ~ 24:00 | 남색 `#000033` | 0.65 → 0.75 (선형 보간) |

### 3-2. 계절별 밝기 보정

| 계절 | 보정 |
|------|------|
| 봄 | 기준 |
| 여름 | 낮 시간 +1시간 (07:00~18:00 완전 밝음) |
| 가을 | 낮 시간 -1시간 (09:00~16:00 완전 밝음) |
| 겨울 | 낮 시간 -2시간 (10:00~15:00 완전 밝음), 최대 어둠 +0.1 |

### 3-3. HUD 시계 표시

화면 우상단에 현재 게임 내 날짜/시간 표시:

```
봄 3일  14:22
```

## 4. 계절별 날씨 시스템

### 4-1. 날씨 결정 방식

매 게임 내 하루가 시작될 때, 그날의 날씨를 결정.
`weather_seed = hash(room_seed + day)` 로 모든 클라이언트가 동일하게 계산.

날씨는 하루 단위로 고정 (하루 중 변경 없음).
단, **폭풍/블리자드**는 하루 안에서 시작/끝 시각이 별도 랜덤 결정.

### 4-2. 계절별 날씨 확률

| 날씨 | 봄 | 여름 | 가을 | 겨울 |
|------|----|------|------|------|
| 맑음 (Clear) | 50% | 40% | 30% | 20% |
| 흐림 (Cloudy) | 25% | 20% | 25% | 30% |
| 비 (Rain) | 20% | 25% | 30% | — |
| 안개 (Fog) | 5% | — | 10% | 10% |
| 낙엽 (Leaves) | — | — | 5% | — |
| 눈 (Snow) | — | — | — | 25% |
| 폭풍 (Storm) | — | 15% | — | — |
| 블리자드 (Blizzard) | — | — | — | 15% |

### 4-3. 날씨별 시각 이펙트

#### 맑음 (Clear)
- 이펙트 없음, 기본 밝기 적용

#### 흐림 (Cloudy)
- 흰/회색 반투명 오버레이 alpha 0.08
- 밝기 약간 감소 (오버레이 alpha +0.05)

#### 비 (Rain)
- 파티클: 흰/연파랑 짧은 선분, 우하향 45°, 빠른 속도
  ```
  quantity: 6/frame
  speed: { x: 80, y: 300 }
  lifespan: 600ms
  alpha: 0.6
  scale: { x: 0.5, y: 2.0 }  ← 세로로 긴 빗방울
  tint: #aaccff
  ```
- 흐림 오버레이 추가 (alpha 0.12)
- 바닥 파문 이펙트: 빗방울이 닿는 곳에 짧은 원형 파문

#### 안개 (Fog)
- 흰색 오버레이 alpha 0.25
- 파티클: 흰 큰 원형, 천천히 수평 이동
  ```
  quantity: 1/frame
  speed: { x: 10~30, y: 0 }
  lifespan: 4000ms
  alpha: { start: 0, peak: 0.2, end: 0 }
  scale: 3.0~6.0
  ```
- 시야 제한 없음 (시각 효과만)

#### 낙엽 (Leaves) — 가을 전용
- 파티클: 주황/빨강/노랑 작은 타원, 나풀나풀 떨어짐
  ```
  quantity: 1~2/frame
  speed: { x: rangeX(-20,20), y: 30~60 }
  lifespan: 3000~5000ms
  rotation: random, angularVelocity: -50~50
  tint: [#ff6600, #cc3300, #ffaa00, #886600]
  scale: 0.4~0.8
  ```

#### 눈 (Snow) — 겨울 전용
- 파티클: 흰 작은 원형, 천천히 낙하
  ```
  quantity: 3~5/frame
  speed: { x: rangeX(-15,15), y: 40~80 }
  lifespan: 4000ms
  alpha: 0.85
  scale: 0.3~0.7
  tint: #ffffff
  ```
- 지면 적설 효과: 흰 오버레이 서서히 증가 (하루 경과마다 alpha +0.05, 최대 0.3)
  - 봄이 되면 서서히 감소

#### 폭풍 (Storm) — 여름 전용
- 비 이펙트 강화 (quantity ×3, speed ×1.5)
- 번개: 5~15초 간격으로 화면 전체 흰 플래시 0.1초
- 천둥 텍스트 이펙트 (시각적 진동): 화면 0.15초 흔들림 (`camera.shake`)
- 어둠 오버레이 추가 (alpha 0.2)

#### 블리자드 (Blizzard) — 겨울 전용
- 눈 이펙트 강화 (quantity ×5, speed ×2, 수평 이동 강화)
- 흰 오버레이 alpha 0.35 (시야 크게 제한)
- 적설 효과 빠르게 증가

## 8. Firebase 동기화 범위

날씨는 `weather_seed` 기반으로 모든 클라이언트가 독립 계산하므로 **Firebase 동기화 불필요**.
게임 시작 타임스탬프만 동기화되면 모든 클라이언트의 날씨가 일치한다.

```
/rooms/{seed}/startTime  ← 게임 생성 시 1회 기록, 이후 변경 없음
```
