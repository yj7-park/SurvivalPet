# 설계 07 — 인벤토리 · 선반 시스템

> **전제 조건**: 06 단계 완료 상태.
> 아이템 드롭/획득, 기본 건설 시스템이 구현되어 있다고 가정한다.

---

## 1. 이번 단계 목표

1. 플레이어 인벤토리 (10칸) 구현
2. `V` 키로 인벤토리 창 토글
3. 아이템 획득 시 인벤토리 슬롯 자동 배정
4. 신규 건설물 **선반** 추가 (20칸)
5. 플레이어 인벤토리 ↔ 선반 아이템 이동

---

## 2. 인벤토리 슬롯 구조

### 2-1. 슬롯 데이터

```typescript
interface InventorySlot {
  itemId: string | null;   // null = 빈 칸
  quantity: number;        // 수량 (0이면 빈 칸으로 처리)
}
```

- 아이템마다 **최대 스택 수량** 존재
- 같은 종류의 아이템은 최대 스택까지 한 슬롯에 쌓임
- 최대 스택 초과 시 다음 빈 슬롯으로 넘어감

### 2-2. 아이템별 최대 스택

| 아이템 | 최대 스택 |
|--------|-----------|
| 목재, 암석, 석재 | 99 |
| 고기, 구운 고기/생선 | 20 |
| 가죽 | 30 |
| 호랑이 이빨 | 10 |
| 활, 나무 칼, 석재 칼 | 1 (장비류) |

---

## 3. 플레이어 인벤토리

### 3-1. 기본 규격

- 슬롯 수: **10칸**
- 정렬: 2행 × 5열
- `V` 키로 열기/닫기 토글
- 인벤토리 열린 상태에서 이동/공격 등 게임 조작 유지

### 3-2. UI 레이아웃

```
┌──────────────────────────────────────────┐
│  🎒 인벤토리                       [V]  │
├──────────────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐    │
│  │ 🪵 │ │ 🪨 │ │ 🐟 │ │    │ │    │    │
│  │ 12 │ │ 5  │ │ 3  │ │    │ │    │    │
│  └────┘ └────┘ └────┘ └────┘ └────┘    │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐    │
│  │ 🗡 │ │    │ │    │ │    │ │    │    │
│  │  1 │ │    │ │    │ │    │ │    │    │
│  └────┘ └────┘ └────┘ └────┘ └────┘    │
├──────────────────────────────────────────┤
│  슬롯: 3/10 사용 중                     │
└──────────────────────────────────────────┘
위치: 화면 중앙 또는 좌측 고정 (추후 확정)
```

### 3-3. 슬롯 상호작용

| 동작 | 결과 |
|------|------|
| 슬롯 호버 | 아이템 이름 + 수량 툴팁 표시 |
| 슬롯 클릭 (무기류) | 해당 무기 장착/해제 |
| 슬롯 클릭 (음식류) | 즉시 섭취 → 허기 회복 |
| 슬롯을 선반으로 드래그 | 아이템 이동 (선반 UI가 열린 경우) |
| 슬롯 우클릭 | 컨텍스트 메뉴 (장착 / 사용 / 버리기) |

### 3-4. 아이템 획득 규칙

1. 바닥 아이템 위를 지나가면 자동 획득
2. 같은 종류 아이템이 인벤토리에 있으면 기존 슬롯에 스택
3. 스택이 가득 찬 경우 다음 빈 슬롯에 배정
4. 인벤토리가 꽉 찬 경우:
   - 아이템을 줍지 못함
   - 슬롯 위에 빨간 "인벤토리 가득참" 알림 1.5초 표시

## 4. 선반 (Shelf)

### 4-1. 건설 정보

| 항목 | 목재 선반 | 석재 선반 |
|------|-----------|-----------|
| 제작 재료 | 목재 ×10 | 석재 ×6 |
| 건설 시간 (STR 5 기준) | 6초 | 12초 |
| 내구도 | 100 | 300 |
| 슬롯 수 | 20칸 | 20칸 |
| 설치 가능 타일 | 흙 | 흙 |

`02-building-system.md`의 건설 목록에 선반 추가.

### 4-3. 선반 아이템 이동 방법

| 동작 | 결과 |
|------|------|
| 내 슬롯 → 선반 슬롯 드래그 | 아이템 이동 |
| 선반 슬롯 → 내 슬롯 드래그 | 아이템 이동 |
| 슬롯 더블클릭 | 반대 인벤토리의 첫 번째 빈 슬롯으로 즉시 이동 |
| [전부 이동 →] 버튼 | 내 인벤토리 전체를 선반으로 이동 (공간 있는 만큼) |

### 4-4. 선반 접근 제한

- **상호작용 거리(2타일) 내**에서만 선반 UI 열 수 있음
- 거리 벗어나면 선반 UI 자동 닫힘
- **선반은 설치한 플레이어 포함 누구나 접근 가능** (공유 수납)
- 멀티플레이 시 다른 플레이어가 같은 선반을 열고 있으면 실시간 동기화

## 6. Firebase 동기화 범위

| 데이터 | 경로 | 비고 |
|--------|------|------|
| 플레이어 인벤토리 | `/rooms/{seed}/players/{id}/inventory` | 본인 클라이언트가 갱신 |
| 선반 인벤토리 | `/rooms/{seed}/maps/{mapX}_{mapY}/shelves/{shelfId}/inventory` | 접근한 클라이언트가 갱신, 실시간 구독 |

- 선반은 여러 플레이어가 동시에 열 수 있으므로 **Firebase 실시간 구독 필수**
- 충돌 방지: 슬롯 단위로 트랜잭션 처리 (같은 슬롯에 동시 쓰기 방지)
