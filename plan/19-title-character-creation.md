# 설계 19 — 메인 화면 & 캐릭터 생성

> **전제 조건**: 01~18 단계 완료 상태.
> SaveSystem, CharacterStats, ProficiencySystem이 구현되어 있다고 가정한다.

---

## 1. 이번 단계 목표

1. **타이틀 화면** — 게임 진입점, 새 게임 / 불러오기 / 설정
2. **Seed 입력** — 새 게임 시 seed 직접 입력 또는 자동 생성
3. **캐릭터 생성** — 능력치 배분, 외형 선택
4. **화면 전환 흐름** — 타이틀 → 캐릭터 생성 → 게임 → 타이틀

## 2. 타이틀 화면

### 2-1. 레이아웃

```
┌──────────────────────────────────────────┐
│                                          │
│           🌲 생존 시뮬레이터 🌲           │
│             (게임 타이틀)                 │
│                                          │
│         [  새 게임 만들기  ]              │
│         [  게임 불러오기  ]              │
│         [     설  정      ]              │
│                                          │
│   버전 v0.19.0          © 2026           │
└──────────────────────────────────────────┘
```

- 배경: seed 없는 랜덤 맵 미리보기 (타이틀 전용 — 상호작용 불가)
  - 맵은 `Math.random()` seed로 생성, 카메라 천천히 패닝 (30초에 맵 1순환)
- 버튼 호버: 글자 밝기 +30%, 커서 pointer

### 2-2. 버튼 동작

| 버튼 | 동작 |
|------|------|
| 새 게임 만들기 | Seed 입력 화면으로 이동 |
| 게임 불러오기 | 저장 슬롯 선택 패널 표시 (plan 17 LoadSlotPanel) |
| 설정 | 설정 패널 표시 |

저장 슬롯이 모두 비어 있으면 "게임 불러오기" 버튼 비활성 (회색 + 툴팁 "저장된 게임이 없습니다").

│                                          │

- 기본값: 자동으로 랜덤 seed 채워져 있음
- [다음 →]: 캐릭터 생성 화면으로 이동

### 4-2. 능력치 배분 규칙

plan 00 게임 플랜 기준:
- **총합 고정: 20**
- 각 능력치 범위: **2 ~ 10**
- 기본 배분: STR 5 / AGI 5 / CON 5 / INT 5 (총합 20)
- [+] / [−] 버튼으로 1씩 조정 (다른 능력치에서 빼와야 올릴 수 있음)

```typescript
// 능력치 조정 규칙
function adjustStat(stats: Stats, target: StatKey, delta: 1 | -1): Stats {
  const newVal = stats[target] + delta;
  if (newVal < 2 || newVal > 10) return stats;          // 범위 초과

  // 총합 고정: 다른 능력치에서 반대 방향으로 조정
  // 올릴 때: 가장 높은 능력치 중 하나에서 −1
  // 내릴 때: 가장 낮은 능력치 중 하나에서 +1
  const others = OTHER_STATS.filter(k => {
    const v = stats[k] + (delta === 1 ? -1 : 1);
    return v >= 2 && v <= 10;
  });
  if (others.length === 0) return stats;                // 조정 불가

  const pivot = delta === 1
    ? others.reduce((a, b) => stats[a] >= stats[b] ? a : b)   // 가장 높은 것
    : others.reduce((a, b) => stats[a] <= stats[b] ? a : b);  // 가장 낮은 것

  return { ...stats, [target]: newVal, [pivot]: stats[pivot] - delta };
}
```

### 4-3. 파생 스탯 미리보기

능력치 변경 시 실시간 업데이트:

| 표시 항목 | 계산식 |
|---------|--------|
| 이동속도 | `120 + (AGI - 5) * 12` px/s |
| 최대 HP | `80 + CON * 8` |
| 낚시 성공률 | `min(95, 40 + AGI*4 + INT*3)` % |
| 벌목 시간 | `max(2, 5 - (STR-5)*0.4)` 초 |
| 채굴 시간 | `max(3, 8 - (STR-5)*0.6)` 초 |

### 4-4. 외형 선택

외형 3종 (SpriteGenerator 기반 팔레트 변경):

| 외형 번호 | 피부색 | 머리카락 | 옷 색 |
|---------|-------|---------|------|
| 1 | #f5cba7 | #4a3728 | #5d8aa8 |
| 2 | #c68642 | #1c1c1c | #8b6914 |
| 3 | #8d5524 | #d4a017 | #556b2f |

[◀] / [▶] 로 순환, 미리보기는 정면 스프라이트 (32×48, 3× 확대 표시).

### 4-5. 이름 입력

- 기본값: "생존자"
- 최대 12자, 빈 칸 불가
- 특수문자 제한: 영문·한글·숫자·공백만 허용

## 5. 게임 시작 흐름

```
[게임 시작!] 클릭
  → 입력 유효성 검사 (이름 비었으면 경고)
  → 저장 슬롯 선택 팝업:
      "어느 슬롯에 저장하시겠습니까?"
      [슬롯 1 (비어 있음)]
      [슬롯 2 (3일차 저장됨 — 덮어씀)]
      [슬롯 3 (비어 있음)]
  → 슬롯 선택 → 덮어쓰기 확인 (plan 17)
  → 로딩 화면 (맵 생성 중…)
  → GameScene 시작 (초기 스폰 위치: 맵 (5,5) 중앙)
```

로딩 화면:
```
┌──────────────────────────────────────────┐
│                                          │
│         🌍 세계를 생성하는 중…            │
│         [██████████░░░░░░░░░░]  55%      │
│         seed: abc123                     │

└──────────────────────────────────────────┘
```

진행 단계:
1. 높이맵 생성 (20%)
2. 타일 분류 (35%)
3. 강·호수 생성 (50%)
4. 나무 배치 (65%)
5. 건설물 초기화 (75%)
6. 시스템 초기화 (90%)
7. 완료 → GameScene 전환 (100%)

## 10. 확정 규칙

- 캐릭터 이름은 저장 데이터(`CharacterSaveData`)에 포함
- 외형 번호(0~2)도 저장 데이터에 포함, 불러오기 시 복원
- Seed는 영문 소문자 + 숫자 1~12자로 정규화 (`seed.toLowerCase().replace(/[^a-z0-9]/g, '')`)
- 빈 Seed 입력 시 자동으로 랜덤 생성 (빈 채로 시작 불가)
- 멀티플레이 진입 시 캐릭터 생성 화면은 동일하게 사용 (Seed가 방 참가 코드로 쓰임)
