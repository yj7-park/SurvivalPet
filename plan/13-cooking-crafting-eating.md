# 설계 13 — 요리 · 제작 · 식사 시스템

> **전제 조건**: 12 단계 완료 상태.
> BuildSystem, InventoryUI, Workbench 패널, Shelf 기반 시스템이 구현되어 있다고 가정한다.

---

## 1. 이번 단계 목표

1. **조리대(kitchen)** 클릭 → 요리 패널 열기, 레시피 선택 → 요리 시작 (진행 바)
2. **작업대(workbench)** 기존 무기 제작에 추가 아이템 레시피 확장
3. **식사** 동작 구체화: 식탁 근처 섭취 시 허기 회복 보너스
4. **선반** 아이템 넣기/꺼내기: 클릭으로 한 개씩 이동

## 2. 요리 시스템

### 2-1. 조리대 상호작용 흐름

```
조리대(kitchen) 좌클릭
  → 범위 내(48px): 요리 패널 즉시 열기
  → 범위 밖: 자동이동 → 도착 후 패널 열기
```

### 2-2. 요리 레시피 목록

| 레시피 | 재료 | 결과물 | 기준 시간 | 능력치 영향 |
|--------|------|--------|----------|-------------|
| 생선 굽기 | 물고기 ×1 | 구운 생선 ×1 | 8초 | INT↑ → 빠름 |
| 고기 굽기 | 날고기 ×1 | 구운 고기 ×1 | 10초 | INT↑ → 빠름 |
| 생선 ×2 굽기 | 물고기 ×2 | 구운 생선 ×2 | 12초 | INT↑ → 빠름 |
| 고기 ×2 굽기 | 날고기 ×2 | 구운 고기 ×2 | 15초 | INT↑ → 빠름 |

요리 시간 계산식 (CharacterStats):
```typescript
get cookTime(): number {
  return Math.max(3000, 8000 - (this.int - 5) * 400);  // ms, 기본 8초
}
```
레시피별 시간 = `cookTime * (레시피 배율)` (×1, ×1.25, ×1.5, ×1.875)

### 2-3. 요리 패널 UI

```
┌─────────────────────────────────┐
│ 🍳 조리대                    [✕] │
├─────────────────────────────────┤
│                                 │
│  [🐟 생선 굽기]  물고기×1       │
│   보유: 3개    → 8.0s    [요리] │
│                                 │
│  [🥩 고기 굽기]  날고기×1       │
│   보유: 0개    → 10.0s   [---] │  (회색, 비활성)
│                                 │
│  [🐟🐟 생선 ×2] 물고기×2       │
│   보유: 3개    → 12.0s   [요리] │
│                                 │
│  [🥩🥩 고기 ×2] 날고기×2       │
│   보유: 0개    → 15.0s   [---] │
│                                 │
│  ████████░░░░  요리 중… 3s     │  (진행 중일 때만)
└─────────────────────────────────┘
```

### 2-4. 요리 진행 규칙

- 한 번에 **1개 레시피**만 진행 가능
- 진행 중 패널 닫기 가능 (요리는 계속 진행)
- 완료 시 결과물 자동으로 인벤토리에 추가
- 인벤토리 가득 참 → 요리 결과물 버려짐 + 경고 표시
- 플레이어가 조리대 48px 초과 이탈 시 → **요리 중단** (재료 소모됨, 결과물 없음)
  - 중단 메시지: "요리가 중단되었습니다"
- 행동(action) +8 회복 (요리 완료 시)

## 3. 제작 시스템 확장

### 3-1. 기존 상태

작업대에서 무기만 제작 가능:
- 활 (목재×8)
- 나무칼 (목재×6)
- 석재칼 (목재×4, 가공석×4)

### 3-2. 추가 레시피

| 제작물 | 카테고리 | 재료 | 기준 시간 |
|--------|---------|------|----------|
| 가공석 ×5 | 재료 | 석재 ×10 | 6초 |
| 낚싯대 | 도구 | 목재 ×3 | 4초 |
| 횃불 | 도구 | 목재 ×2 | 2초 |

> 낚싯대: 낚시 성공률 보정 +20% (아이템 착용 개념은 추후 설계)  
> 횃불: 야간 시야 개선 (추후 설계)  
> 가공석: 석재 건설 시 필요한 재료

### 3-3. 작업대 패널 UI 개선

기존 무기 목록에 **카테고리 탭** 추가:

```
┌─────────────────────────────────┐
│ 🔨 작업대                    [✕] │
├─────────────────────────────────┤
│ [무기] [재료] [도구]            │  ← 탭
├─────────────────────────────────┤
│ 아이템 목록 (기존과 동일 형태)  │
└─────────────────────────────────┘
```

제작 시간 계산식:
```typescript
get craftTime(): number {
  return Math.max(2000, 6000 - (this.int - 5) * 400);  // ms, 기본 6초
}
```
레시피별 시간 = `craftTime * 레시피 배율`

### 3-4. 제작 규칙

- 한 번에 1개만 제작 가능
- 제작 중 패널 닫아도 진행 계속
- 작업대 범위 이탈 시 → **제작 취소**, 재료 환불 없음
- 행동(action) +12 회복 (제작 완료 시)

## 4. 식사 시스템 구체화

### 4-1. 현재 동작

인벤토리(V키) → 음식 클릭 → 즉시 허기 회복

### 4-2. 변경 후: 식탁 유무에 따른 차이

| 상황 | 허기 회복 | 연출 |
|------|----------|------|
| **식탁 근처(48px 이내)** 식사 | 기본값 × **1.3** (30% 보너스) | "🍽 식탁에서 먹었습니다" 팝업 |
| **바닥에서** 식사 (식탁 없음) | 기본값 × 1.0 | 별도 연출 없음 |

### 4-3. 허기 회복 기본값

| 음식 | 기본 회복량 |
|------|------------|
| 물고기 (날것) | 15 |
| 구운 생선 | 25 |
| 날고기 | 10 |
| 구운 고기 | 35 |

기존 InventoryUI의 `FOOD_RESTORE` 상수 참조. 보너스 계산 후 소수점 올림.

### 4-4. 식탁 인접 판정

InventoryUI의 `eat()` 호출 시 GameScene에 query:

```typescript
// GameScene에 추가
isNearTable(playerX: number, playerY: number): boolean {
  for (const struct of buildSystem.getAllStructures()) {
    if (struct.defName === 'table') {
      const wx = struct.tileX * TILE_SIZE + TILE_SIZE / 2;
      const wy = struct.tileY * TILE_SIZE + TILE_SIZE / 2;
      if (Math.hypot(playerX - wx, playerY - wy) <= 48) return true;
    }
  }
  return false;
}
```

InventoryUI 생성자에 `getNearTable: () => boolean` 콜백 추가.

### 4-5. 식사 연출 (식탁 보너스 시)

```
"🍽 식탁에서 먹었습니다 (+30%)" 텍스트
  → 화면 중앙 하단 고정 위치에 1.5초 표시 후 페이드아웃
  → 색상: #ffd700 (골드)
  → 폰트: 12px monospace
```

## 5. 선반 상호작용 개선

### 5-1. 현재 ShelfUI 상태

- 플레이어 인벤토리(좌) / 선반(우) 2열 표시
- "전부 이동" 버튼 (내 인벤 → 선반 전부 이동)
- **클릭으로 아이템 이동 미구현**

### 5-2. 추가 기능

#### 클릭 상호작용

| 동작 | 결과 |
|------|------|
| 플레이어 인벤 아이템 좌클릭 | 1개 선반으로 이동 |
| 플레이어 인벤 아이템 우클릭 | 전부 선반으로 이동 |
| 선반 슬롯 좌클릭 | 1개 플레이어 인벤으로 이동 |
| 선반 슬롯 우클릭 | 해당 슬롯 전부 플레이어 인벤으로 이동 |

#### "1개 이동" 로직

```
플레이어 인벤 아이템 A (예: 목재 ×20) 좌클릭
  → 선반 addItem('item_wood', 1)
  → 성공 시 플레이어 인벤 remove('item_wood', 1)
  → ShelfUI 갱신
```

#### 선반 → 플레이어 인벤 이동 제약

- 플레이어 인벤토리가 가득 찼으면 이동 불가
- 이동 불가 시 슬롯 빨간 테두리 + 툴팁 "인벤토리 가득 참"

### 5-3. 슬롯 마우스오버 표시

```
마우스오버:
  슬롯 배경색 밝아짐 (#2a3040)
  하단 툴팁: "[아이템명] ×수량 / 클릭: 1개 이동 / 우클릭: 전부 이동"
```

### 5-4. 빠른 이동 버튼 확장

현재 "→ 전부 이동" 버튼 외:

```
[→ 전부 이동] [← 전부 꺼내기]
```

- **전부 꺼내기**: 선반의 모든 아이템 → 플레이어 인벤 (인벤 용량 초과분은 선반에 유지)

## 7. 확정 규칙

### 요리
- 요리 중 이탈 시 취소, 재료 소모됨 (경고 메시지 표시)
- 인벤 가득 참 상태에서 완료 시 결과물 소멸 + 경고

### 제작
- 제작 중 이탈 시 취소, 재료 소모됨
- 가공석 제작은 실시간 진행 바 표시 (무기 제작과 동일)

### 식탁 보너스
- 식탁이 여러 개 있어도 1개만 인접하면 보너스 적용
- 침대/의자는 식탁 보너스 없음

### 선반
- 플레이어 이동으로 범위 이탈 시 자동 닫힘 (현재 구현대로 유지)
- 이동 중에는 ShelfUI의 playerPos 실시간 업데이트 필요 (GameScene.update()에서 호출)
