## 2. 피로 수치 설계

### 2-1. 기본 수치 (plan 00 확정값)

| 항목 | 값 |
|------|-----|
| 범위 | 0 ~ 100 |
| 초기값 | 100 |
| 자연 감소 (게임 내 1일 기준) | −50 |
| 현실 시간 감소율 | −50 / 1800초 = **약 −0.0278/초** |

```typescript
// 실시간 감소 (delta: ms)
fatigue -= (50 / 1_800_000) * delta;
```

### 2-2. 활동별 추가 피로 감소

| 활동 | 추가 감소량 |
|------|-----------|
| 벌목 1회 | −3 |
| 채굴 1회 | −4 |
| 낚시 1회 | −1 |
| 전투 공격 1회 | −2 |
| 건설 완료 1회 | −5 |
| 달리기(미래 확장) | −예약 |

CON 능력치 보정:
```typescript
// CON 높을수록 활동 피로 감소량 줄어듦
const conMult = Math.max(0.5, 1.0 - (stats.con - 5) * 0.06);
// CON 2 → 1.18×  / CON 5 → 1.0×  / CON 10 → 0.7×
fatigueDelta *= conMult;
```

## 3. 수면 진입

### 3-1. 수면 가능 조건

| 조건 | 내용 |
|------|------|
| 침대 상호작용 범위 | 48px 이내 |
| 피로 수치 | 제한 없음 (피로 100 이어도 잘 수 있음) |
| 시간대 제한 | **없음** — 언제든 수면 가능 |
| 전투 중 | 불가 — "전투 중에는 잘 수 없습니다" |
| 광란 상태 | 불가 (조작 불가 상태) |

### 3-2. 침대 클릭 흐름 (일반 수면)

```
침대 클릭
  → 수면 확인 팝업:
    "수면하시겠습니까?
     기상 조건: 피로 70 이상 회복 또는 게임 내 아침(06:00)"
      [수면]   [취소]
  → [수면] 선택 → SleepSystem.startSleep()
```

### 3-3. 침대 종류별 수면 효율

| 침대 종류 | 시간당 피로 회복 | 비고 |
|---------|--------------|------|
| 목재 침대 | 60 / 게임일 | 기본 |
| 석재 침대 | 75 / 게임일 | 연구 해금 필요 (plan 14) |
| 맨바닥 수면 (강제) | 30 / 게임일 | 침대 없이 강제 수면 시 |

실내(지붕 아래) 수면 보너스: 회복량 **+20%**

```typescript
function getSleepRecoveryRate(bedType: BedType | null, isIndoor: boolean): number {
  const base: Record<BedType, number> = {
    bed_wood:  60,
    bed_stone: 75,
    none:      30,   // 강제 수면
  };
  const rate = base[bedType ?? 'none'];
  return isIndoor ? rate * 1.2 : rate;
}
// 단위: 피로 회복량 / 게임 내 1일
```

## 4. 수면 중 시스템

### 4-1. 시간 가속

수면 중 게임 시간 **4배속** 진행:
```typescript
const SLEEP_TIME_MULT = 4.0;
gameTime.update(delta * SLEEP_TIME_MULT);
```

결과: 현실 7.5분(게임 내 6시간)이 약 **1분 52초** 만에 경과.

### 4-2. 수면 중 상태 변화

| 항목 | 수면 중 처리 |
|------|-----------|
| 피로 | 침대 효율에 따라 회복 (시간 가속 반영) |
| 허기 | 정상 감소 (시간 가속 반영 — 빠르게 소모) |
| 행동 수치 | 감소 없음 (수면 = 활동으로 인정) |
| HP 자연 회복 | 없음 (수면 중 HP 회복은 미지원) |
| 적 스폰 | 정상 작동 (수면 중 기습 가능) |
| 광란 | 수면 중 행동 수치 감소 없으므로 진입 안 함 |

### 4-3. 수면 중단 조건 (강제 기상)

| 조건 | 처리 |
|------|------|
| 피격 (HP 감소) | 즉시 기상, "공격을 받아 잠에서 깼습니다!" |
| 허기 0 도달 | 즉시 기상, "배고파서 잠에서 깼습니다!" |
| 사용자 ESC 키 | 기상 확인 팝업 → 확인 시 기상 |

### 4-4. 정상 기상 조건 (둘 중 하나)

1. **피로 수치 ≥ 70** 도달
2. **게임 내 시간 06:00** (아침) 도달 — 피로가 덜 찼어도 기상

## 5. 강제 수면 (피로 0)

```
피로 수치 = 0
  → 경고 메시지: "극도로 피곤합니다... 쓰러집니다"
  → 1초 후 강제 수면 진입
    → 플레이어 조작 완전 차단
    → 현재 위치에서 바로 수면 (침대 없음)
    → 맨바닥 수면 효율 적용 (30/일)
    → 기상 조건: 피로 ≥ 40 (일반 수면 70보다 낮음)
```

강제 수면 시각 연출:
```
화면 전체 점진적 어두워짐 (2초)
캐릭터 스프라이트 → 누운 자세 (정면 스프라이트 회전 90°)
화면 중앙: "💤 수면 중..."  (회색, 16px)
```

## 8. SleepSystem 클래스

```typescript
export class SleepSystem {
  private sleeping: boolean = false;
  private forcedSleep: boolean = false;
  private bedRef: Building | null = null;

  // 수면 시작 (침대 클릭 또는 강제)
  startSleep(bed: Building | null, forced: boolean): void

  // 매 프레임: 피로 회복, 기상 조건 체크
  update(delta: number, stats: CharacterStats, gameTime: GameTime): SleepUpdateResult

  // 기상 처리
  wakeUp(reason: WakeReason): void

  // 상태 조회
  isSleeping(): boolean
  isForcedSleep(): boolean

  // 저장용
  serialize(): { sleeping: boolean; fatigue: number }
}

type WakeReason = 'recovered' | 'morning' | 'attacked' | 'starving' | 'user';

type SleepUpdateResult = {
  woke?: WakeReason;
  fatigueRecovered: number;
};
```

## 10. 확정 규칙

- 수면 중 저장 시 `sleeping: false` 로 기록 — 재개 시 깨어있는 상태로 시작
- 피로가 이미 70 이상일 때 침대 클릭 → "충분히 쉬었습니다" 안내 (수면 불가)
- 강제 수면 중 피격 → 즉시 기상 (맨바닥이므로 일반 기상보다 더 취약)
- 허기가 0인 상태에서 수면 시도 → "배가 너무 고파 잠들 수 없습니다" (수면 불가)
- 수면 중 허기 감소는 시간 가속(4배) 반영 — 자기 전 음식 섭취 권장 유도
- 멀티플레이에서 수면 중 캐릭터는 다른 플레이어에게 누운 자세로 표시
