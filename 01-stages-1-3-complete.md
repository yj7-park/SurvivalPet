# 웹 기반 2D 생존 시뮬레이션 게임 개발 계획 v1.1

> **v1.1 기준일: 2026-04-08**
> 1~3단계 구현 완료 기준 스냅샷

## 게임 개요

- 장르: 2D 실시간 멀티플레이 생존 시뮬레이션
- 플레이어는 개인 캐릭터를 조작하여 지정된 맵 안에서 생존
- Seed 값 기반으로 게임 공유 가능 (같은 Seed = 같은 맵/세계)
- 키보드로 캐릭터 조작

## 기술 스택

### 프론트엔드
- **언어**: TypeScript
- **게임 프레임워크**: Phaser.js (2D 게임 전용, 키보드 입력/스프라이트/물리 내장)
- **빌드 툴**: Vite 5
- **Seed 기반 난수**: `seedrandom` 라이브러리
- **지형 노이즈**: `simplex-noise` 라이브러리
- **호스팅**: Cloudflare Pages (무료)

### 백엔드 / 실시간 동기화
- **서비스**: Firebase Realtime Database (무료 플랜)
- **역할**: 별도 WebSocket 서버 없이 플레이어 상태 실시간 동기화
- **무료 플랜 한도**: 동시 접속 100명, DB 1GB

## 아키텍처

```
[브라우저 - Phaser.js]
    │
    │  Seed 값으로 맵 생성 (모든 클라이언트가 동일하게 계산)
    │  플레이어 입력 → Firebase에 상태 업데이트
    │  Firebase 변경 감지 → 다른 플레이어 위치/상태 반영
    │
[Firebase Realtime Database]
    ├── /rooms/{seed}/players/{playerId}
    │       ├── x, y (위치)
    │       ├── hp (체력)
    │       └── ...
    └── /rooms/{seed}/state (게임 전체 상태)
```

## Seed 시스템

- 새 게임 생성 시 랜덤 Seed 값 자동 발급 (ex: `abc123`)
- Seed → `seedrandom` 난수 생성기 → 맵/지형/이벤트 결정
- 같은 Seed를 입력한 모든 클라이언트는 동일한 맵을 독립 계산으로 생성
- Firebase에서 같은 Seed의 방(room)에 접속하여 실시간 동기화

## 게임 컨텐츠 및 조작

### 시간 시스템
- 현실 시간 30분 = 게임 내 1일
- 1일 = 86,400 게임 내 초 → 현실 1초 ≈ 게임 내 48초
- 낮/밤 사이클, 날짜 경과에 따른 이벤트 트리거 가능

```
현실 시간  │  게임 내 시간
──────────────────────────
  0분      │  0일 00:00 (새벽)
  7.5분    │  0일 06:00 (아침)
 15분      │  0일 12:00 (정오)
 22.5분    │  0일 18:00 (저녁)
 30분      │  1일 경과
```

### 캐릭터 조작

#### 이동 (키보드)
- `↑ ↓ ← →` 방향키 **또는 WASD** 로 캐릭터 이동 ✅ 구현 완료
- 대각선 이동 가능 (두 키 동시 입력, 속도 정규화 ×0.7071) ✅ 구현 완료

#### 상호작용 (마우스)
- 대상 클릭 → 해당 대상과 상호작용
- **상호작용 가능 거리** 이내: 즉시 상호작용 실행
- **상호작용 가능 거리** 초과 시:
  - 자동으로 대상을 향해 최단 경로 이동 시작
  - 캐릭터가 거리 내에 도달하면 자동으로 상호작용 실행
  - 이동 중 다른 키 입력 시 자동 이동 취소

#### 경로 탐색 (Auto-pathfinding)
- 알고리즘: **A\* (A-star)**
- 장애물(벽, 나무, 물 등) 우회하여 최단 경로 계산
- 경로는 타일 단위로 계산, 이동은 부드럽게 보간(lerp)

### 캐릭터 능력치 시스템

#### 4가지 기본 능력치

| 능력치 | 영향 범위 |
|--------|-----------|
| **힘 (STR)** | 벌목 속도, 전투 공격력, 허기 감소 속도 |
| **민첩성 (AGI)** | 이동 속도, 낚시 성공률, 회피 (추후 전투) |
| **체력 (CON)** | 최대 HP, 피로 감소 속도, HP 회복 속도 |
| **지능 (INT)** | 낚시 성공률, 아이템 획득 효율, 추후 기술/제작 |

#### 능력치 생성 규칙
- **총합 고정: 20**
- 각 능력치 범위: **2 ~ 10**
- 캐릭터 생성 시 랜덤 분배 (Seed 기반)

```
생성 알고리즘:
1. 각 능력치를 최솟값 2로 초기화 (합계 8)
2. 남은 포인트 12를 랜덤하게 4개 능력치에 분배
3. 각 능력치가 10을 초과하지 않도록 제한
```

#### 능력치 → 파생 스탯 공식

**이동 속도**
```
base_speed = 120 (px/초)
move_speed = base_speed + (AGI - 5) * 12
// AGI 2 → 84px/s  |  AGI 5 → 120px/s  |  AGI 10 → 180px/s
```

**최대 HP**
```
base_hp = 80
max_hp = base_hp + CON * 8
// CON 2 → 96HP  |  CON 5 → 120HP  |  CON 10 → 160HP
```

**허기 감소 속도** (게임 내 1일 기준)
```
hunger_decay = 40 - (STR - 5) * 2
// STR 2 → -46/일  |  STR 5 → -40/일  |  STR 10 → -30/일
```

**피로 감소 속도** (게임 내 1일 기준)
```
fatigue_decay = 50 - (CON - 5) * 3
// CON 2 → -59/일  |  CON 5 → -50/일  |  CON 10 → -35/일
```

#### 능력치 → 상호작용 영향

**벌목 소요 시간**
```
log_time = 5 - (STR - 5) * 0.4  (초, 최소 2초)
// STR 2 → 6.2초  |  STR 5 → 5초  |  STR 10 → 3초
```

**낚시 성공률**
```
fish_rate = 40 + AGI * 4 + INT * 3  (%, 최대 95%)
// AGI2·INT2 → 54%  |  AGI5·INT5 → 75%  |  AGI10·INT10 → 110% → cap 95%
```

**낚시 소요 시간**
```
fish_time = 10 - (AGI - 5) * 0.6  (초, 최소 5초)
// AGI 2 → 11.8초  |  AGI 5 → 10초  |  AGI 10 → 7초
```

### 캐릭터 상태 조건

캐릭터는 **체력(HP)** 과 아래 3가지 수치를 유지해야 한다.

#### 체력 (HP)
- 범위: 0 ~ 최대 HP (최대 HP는 debuff에 의해 감소 가능)
- 전투 시스템에서 핵심 수치로 사용
- 최대 HP가 줄어도 현재 HP가 그 이상이면 최대치에 맞춰 깎임

#### 3가지 상태 수치

| 조건 | 감소 원인 | 회복 방법 |
|------|-----------|-----------|
| **허기** | 시간 경과, 활동 | 음식 섭취 |
| **피로** | 활동, 시간 경과 | 수면 (특정 장소/시간) |
| **행동** | 아무 행동도 하지 않는 시간 | 다양한 행동 수행 |

#### 수치 설계 (초안)
- 각 수치는 **0 ~ 100** 범위
- 게임 내 1일(현실 30분) 동안 자연 감소량 예시:
  - 허기: -40 / 일 (음식 2~3회 필요)
  - 피로: -50 / 일 (수면 필수)
  - 행동: -20 / 일 (꾸준한 활동 필요)

#### 수치 0 도달 시 페널티

| 조건 | 페널티 |
|------|--------|
| **허기 0** | 시간 경과에 따라 최대 HP 감소 debuff 누적 / 허기 > 0 이 되면 debuff가 시간에 따라 감소 (허기 수치가 높을수록 감소 속도 빠름) |
| **피로 0** | 강제 수면 상태 돌입 → 플레이어 조작 불가, 일정 시간 경과 후 피로 회복되면 해제 |
| **행동 0** | 광란 모드 30초 돌입 → 주변 캐릭터/사물 자동 공격, 플레이어 조작 완전 불가 → 30초 후 행동 수치 일부 회복되며 해제 |

- 완료 시 나무 오브젝트 제거 → 해당 타일 흙으로 복원
- 목재 ×20 인벤토리 추가
- 일정 시간 후 나무 재생성 (시간 미정)

#### 낚시
- 물 타일에 인접한 흙 타일에서만 가능
- 완료 시 성공/실패 판정 (AGI + INT 기반 확률)
  - 성공: 물고기 ×1 인벤토리 추가
  - 실패: 산출물 없음

#### 암반 채굴
- 암반 타일 클릭 → STR 기반 소요 시간 프로그레스
  ```
  mine_time = 8 - (STR - 5) * 0.6  (초, 최소 3초)
  // STR 2 → 9.8초  |  STR 5 → 8초  |  STR 10 → 5초
  ```
- 완료 시 암석(아이템) ×10 인벤토리 추가
- 채굴된 암반 타일은 흙으로 변환 (재생성 없음 — 영구 소모 자원)

#### 행동 수치 연동
- 벌목/채굴/낚시 수행 시 **행동 수치 회복**
- 같은 액션 반복 시 회복량 감소 (단순 반복 방지)

## 맵 설계

### 월드 구조

```
전체 월드: 10×10 맵 = 100개의 맵
각 맵: 100×100 타일
전체 타일 수: 1,000×1,000 = 1,000,000 타일

[맵 좌표 (0,0)] [맵 (1,0)] ... [맵 (9,0)]
[맵 (0,1)     ] [맵 (1,1)] ... [맵 (9,1)]
...
[맵 (0,9)     ]             ... [맵 (9,9)]
```

- 맵 끝 타일 도달 시 인접 맵으로 자동 이동 ✅ 구현 완료
- 각 맵은 필요 시 로드 (전체를 한 번에 메모리에 올리지 않음) ✅ 구현 완료

### 타일 종류

| 타일 | 이동 가능 | 설명 |
|------|-----------|------|
| **흙** | O | 기본 지형, 나무 생성 가능 |
| **물** | X | 강/호수, 이동 불가 (보트 등 추후 추가 가능) |
| **암석** | X | 산지/암반, 이동 불가 |
| **나무** | X | 흙 위에 배치된 오브젝트, 채집 가능 |

### 맵 생성 알고리즘

#### 1단계: 높이맵 생성 (Simplex Noise) ✅ 구현 완료

- `simplex-noise` 라이브러리 사용, seed 값으로 초기화
- 각 맵(mapX, mapY)은 노이즈 공간에서 오프셋 좌표 사용
  → 인접한 맵 간 지형이 자연스럽게 이어짐

```
noiseX = (mapX * 100 + tileX) / 스케일
noiseY = (mapY * 100 + tileY) / 스케일
height = simplex.noise2D(noiseX, noiseY)  // -1 ~ 1
```

- **멀티 옥타브(Octave)** 적용으로 세부 지형 표현:
  ```
  height = noise(x, y) * 1.0         // 큰 지형 윤곽
           + noise(x*2, y*2) * 0.5   // 중간 굴곡
           + noise(x*4, y*4) * 0.25  // 세부 질감
  ```

#### 2단계: 높이값 → 타일 분류 ✅ 구현 완료

```
height < -0.15          → 물 (호수/강 후보)
-0.15 <= height < 0.55  → 흙
height >= 0.55          → 암석
```

#### 3단계: 강 생성 ✅ 구현 완료

1. 높이맵에서 암석 지역의 고점(peak)을 여러 개 선정
2. 각 고점에서 **경사 하강법(Steepest Descent)** 으로 경로 추적
3. 경로를 따라 타일을 물로 변환 (폭: 1~2 타일)
4. 강이 지나는 흙 타일의 인접 타일을 부분적으로 물로 확장

#### 4단계: 호수 보정 (Cellular Automata) ✅ 구현 완료

- 규칙: 인접 8칸 중 물 타일이 5개 이상이면 물로 변환 (3회 반복)
- 5타일 미만 물 덩어리 → 흙으로 되돌림

#### 5단계: 암석 보정 ✅ 구현 완료

- 인접 암석 타일이 2개 미만인 고립 암석 → 흙으로 변환

#### 6단계: 나무 배치 (Poisson Disk Sampling) ✅ 구현 완료

- 흙 타일에만 배치, 최소 간격 2타일
- 밀도: 흙 타일의 약 20% (seed 기반 랜덤)

#### 화면에 보이는 범위

- 초기 줌: 2.0 (타일 표시 크기 64px)

#### 카메라
- 카메라는 플레이어 캐릭터를 중심으로 따라다님 ✅
- 맵 끝 부근에서 카메라 경계 클램프 ✅

#### 맵 전환 ✅ 구현 완료
- 캐릭터가 맵 끝 벗어나면 인접 맵으로 전환
- 전환 시: 현재 맵 RenderTexture 파괴 → 새 맵 생성 및 렌더링
- 인접 4방향 맵을 100ms 딜레이 후 백그라운드 프리로드

```
[맵 전환 흐름]
캐릭터가 오른쪽 끝 도달
  → 현재 맵 (2,1) RenderTexture 파괴
  → 미리 생성된 맵 (3,1) RenderTexture 신규 생성
  → 백그라운드에서 새 인접 맵들 생성 시작
```

#### 렌더링 구현 방식 ✅
- `Phaser.GameObjects.RenderTexture` 사용 (드로우콜 최소화)
- **1패스**: 전체 지면 타일 (흙/물/암석, 나무 위치엔 흙) 스탬핑
- **2패스**: 나무 오브젝트 y순 스탬핑 (32×48, 타일 위로 16px 오버행)
- 플레이어 스프라이트 depth=1, RenderTexture depth=0

## 스프라이트 및 그래픽

### 방침
- 외부 이미지 파일(PNG 등) 사용하지 않음
- 모든 스프라이트는 **Canvas API 코드로 직접 생성**
- 레퍼런스: `sprite-generator.html` (브라우저에서 미리보기 및 PNG 저장 가능)
- Phaser 로드 시 `create()` 단계에서 `registerTextures(scene)` 호출하여 텍스처 등록

### 스프라이트 목록 (구현 완료)

| 스프라이트 | 크기 | 텍스처 키 | 설명 |
|------------|------|-----------|------|
| 흙 (Dirt) | 32×32 | `tile_dirt` | 갈색 베이스, 노이즈 질감, 작은 돌멩이 |
| 물 (Water) | 32×32 | `tile_water` | 파란색 베이스, 깊이감, 반짝임 |
| 암석 (Rock) | 32×32 | `tile_rock` | 회색 베이스, 균열 텍스처 |
| 나무 (Tree) | 32×48 | `obj_tree` | 그림자 + 줄기 + 잎 3레이어, 투명 배경 |
| 캐릭터 정면 | 32×32 | `char_down` | 4방향 픽셀아트 |
| 캐릭터 뒤 | 32×32 | `char_up` | |
| 캐릭터 좌 | 32×32 | `char_left` | |
| 캐릭터 우 | 32×32 | `char_right` | 좌의 좌우 반전 |

### Phaser 적용 방식 (구현 완료)
```typescript
// src/world/SpriteGenerator.ts
export function registerTextures(scene: Phaser.Scene): void {
  scene.textures.addCanvas('tile_dirt',  drawDirt());
  scene.textures.addCanvas('tile_water', drawWater());
  scene.textures.addCanvas('tile_rock',  drawRock());
  scene.textures.addCanvas('obj_tree',   drawTree());
  scene.textures.addCanvas('char_down',  drawCharacter('down'));
  scene.textures.addCanvas('char_up',    drawCharacter('up'));
  scene.textures.addCanvas('char_left',  drawCharacter('left'));
  scene.textures.addCanvas('char_right', drawCharacter('right'));
}
```
