# 설계 06 — 적군 침입 시스템

> **전제 조건**: 05 단계 완료 상태.
> 무기 제작, 자동 연속 공격, 회피 시스템이 구현되어 있다고 가정한다.

---

## 1. 이번 단계 목표

1. 하루 0~2회 랜덤 적군 침입 이벤트 구현
2. 침입 알림 UI (전체 일시정지 + 확인 패널)
3. 위험 상태 시각 이펙트 (빨간 테두리 + 불꽃 효과)
4. 적군 AI 및 스탯/아이템 랜덤 생성
5. 적군 사망 시 아이템 드롭 + 위험 상태 해제

---

## 2. 침입 발생 규칙

### 2-1. 발생 빈도

```
게임 내 1일 = 현실 30분

하루 내 침입 횟수: 0~2회 (균등 확률)
  - 0회: 33%
  - 1회: 33%
  - 2회: 33%

침입 발생 시각: 게임 내 시간 기준 랜덤
  - 06:00 ~ 22:00 사이에만 발생 (야간 침입 없음)
  - 2회 침입 시 최소 간격: 게임 내 4시간 (현실 6분)
```

### 2-2. 랜덤 결정 항목

침입 발생 시 아래 항목을 Seed + 날짜 + 침입 인덱스 기반으로 결정.
**모든 클라이언트가 동일한 결과를 계산** (Firebase 통신 최소화).

```
invasion_seed = hash(room_seed + day + invasion_index)
```

| 항목 | 범위 | 비고 |
|------|------|------|
| 적군 수 | 2~8명 | invasion_seed 기반 |
| 각 적 STR | 2~10 | 개별 랜덤 |
| 각 적 AGI | 2~10 | 개별 랜덤 (총합 20 규칙 적용) |
| 각 적 CON | 2~10 | 개별 랜덤 |
| 각 적 INT | 2~10 | 개별 랜덤 |
| 각 적 무기 | 맨손/나무칼/석재칼/활 | 가중치 랜덤 |
| 각 적 소지 아이템 | 0~3종 | 아이템 풀에서 랜덤 |
| 맵 진입 방향 | 상/하/좌/우 중 1개 | 랜덤 |

### 2-3. 적군 무기 가중치

| 무기 | 가중치 |
|------|--------|
| 맨손 | 30% |
| 나무 칼 | 35% |
| 석재 칼 | 20% |
| 활 | 15% |

### 2-4. 적 소지 아이템 풀

적 사망 시 드롭되는 아이템. 침입 시 각 적에게 0~3종 랜덤 지급.

| 아이템 | 드롭 가중치 |
|--------|-------------|
| 고기 (raw_meat) | 높음 |
| 목재 (item_wood) | 높음 |
| 암석 (item_stone) | 중간 |
| 가죽 (item_hide) | 중간 |
| 석재 (item_processed_stone) | 낮음 |
| 호랑이 이빨 (item_tiger_fang) | 매우 낮음 |

## 3. 침입 알림 및 일시정지

### 3-1. 흐름

```
[침입 발생 시각 도달]
        │
        ▼
Firebase에 침입 이벤트 기록
  /rooms/{seed}/invasions/{day}_{index}
    ├── triggered: true
    └── acknowledgedBy: {}
        │
        ▼
모든 클라이언트: 침입 감지
        │
        ▼
게임 일시정지 (모든 이동/AI/타이머 중단)
        │
        ▼
침입 알림 패널 표시
        │
        ├── 각 플레이어 확인 버튼 클릭 or 엔터
        │       → Firebase acknowledgedBy에 자신 ID 기록
        │
        └── 15초 경과 (타이머)
                │
                ▼
        게임 재개 (일시정지 해제)
        적군 맵 진입 방향에 스폰
```

### 3-2. 침입 알림 패널 UI

```
┌─────────────────────────────────────────┐
│  ⚠  적군이 침입했습니다!               │
│                                         │
│  방향: 북쪽                             │
│  규모: 5명                              │
│                                         │
│  [준비됨 ✓] 홍길동                     │  ← 확인한 플레이어
│  [ 대기 중 ] 김철수                    │  ← 미확인 플레이어
│                                         │
│  자동 재개까지: 12초                    │  ← 카운트다운
│                                         │
│        [ 확인 (Enter) ]                 │
└─────────────────────────────────────────┘
위치: 화면 중앙
배경: 반투명 검정 (배경 흐림 처리)
테두리: 빨간색 2px
```

## 4. 위험 상태 시각 이펙트

적군이 살아있는 동안 화면에 위험 상태를 지속 표시.

### 4-1. 빨간 테두리 펄스

- 화면 4면 테두리에 빨간 그라디언트 오버레이
- 1.2초 주기로 투명도 사인 곡선 애니메이션

```
opacity = 0.3 + 0.3 * sin(time * 2π / 1.2)
// 0.0 ~ 0.6 범위로 부드럽게 진동
```

- 구현: Phaser Graphics 오브젝트를 camera-fixed로 배치
  - 상단: 0,0 ~ screenW, 60 (그라디언트: 위=빨강, 아래=투명)
  - 하단: 0, screenH-60 ~ screenW, screenH
  - 좌/우: 동일 방식

### 4-2. 불꽃/이글이글 효과

- 화면 4면 테두리를 따라 파티클 이펙트
- Phaser의 ParticleEmitter 사용

```
파티클 설정:
  texture: 작은 주황/노랑 원형 (8×8 px, 코드 생성)
  emitZone: 화면 테두리 4면 (선형)
  quantity: 2~4개 / 프레임
  lifespan: 600~1000ms
  speed: { x: rangeX(-20, 20), y: -40 ~ -80 }
  scale: { start: 0.8, end: 0 }
  alpha: { start: 0.9, end: 0 }
  tint: [0xff4400, 0xff8800, 0xffcc00]  ← 주황~노랑
  blendMode: ADD  (가산 혼합으로 빛나는 느낌)
```

### 4-3. 위험 상태 HUD 텍스트

- 화면 상단 중앙에 텍스트 표시

```
⚠ 적군 침입 중 — 남은 적: 5명
```

- 텍스트 색상: 빨간색, 1초 주기로 깜빡임

## 5. 적군 AI

적군은 동물 AI와 동일한 상태 머신 구조를 사용하되,
기본 상태가 **AGGRESSIVE** 이며 항상 가장 가까운 플레이어를 추격한다.

### 5-1. 상태 전이

```
스폰
  │
  ▼
[AGGRESSIVE]  ──────────────────────────────┐
  │  가장 가까운 플레이어 추격              │
  │  사거리 내 진입 → 공격                 │
  │  공격 쿨다운마다 반복                  │
  │                                         │
  └── 모든 플레이어 사망/로그아웃          │
        → [WANDER] (맵 내 배회)             │
              플레이어 재접속 시 다시 ──────┘
                AGGRESSIVE 복귀
```

### 5-2. 적군 스탯 파생 (동물과 동일 공식 적용)

```
max_hp     = 80 + CON * 8
move_speed = 120 + (AGI - 5) * 12  px/s
atk_damage = weapon_base + STR * 2
atk_cooldown = weapon_base_cooldown - (AGI - 5) * 0.1  (최소 0.6초)
dodge_chance = AGI * 4%  (최대 40%)
```

### 5-3. 맵 진입

- 침입 방향 기준 맵 테두리에서 스폰
- 예: 북쪽 침입 → 맵 상단 y=0 라인, x 위치는 랜덤하게 분산

---

## 6. 적군 사망 처리

1. HP 0 → 0.5초 페이드 아웃
2. 소지 아이템 전부 해당 타일에 드롭
3. 적군 객체 제거
4. Firebase에 해당 적 사망 기록
   ```
   /rooms/{seed}/invasions/{day}_{index}/enemies/{enemyId}/dead: true
   ```
5. **맵 내 모든 적 사망 시**:
   - 위험 이펙트 (테두리 펄스 + 파티클) 즉시 제거
   - HUD 텍스트 제거
   - 중앙 패널 표시: "✓ 적군을 모두 물리쳤습니다!" (3초 후 자동 닫힘)

---

## 7. Firebase 동기화 범위

| 데이터 | 경로 | 동기화 주체 |
|--------|------|-------------|
| 침입 이벤트 발생 | `/invasions/{day}_{index}/triggered` | 첫 번째 감지 클라이언트 |
| 플레이어 확인 | `/invasions/{day}_{index}/acknowledgedBy/{playerId}` | 각 클라이언트 본인 |
| 적 사망 | `/invasions/{day}_{index}/enemies/{id}/dead` | 최후 타격 클라이언트 |

- 적군 위치/이동은 각 클라이언트가 독립 계산 (동기화 없음)
- invasion_seed가 동일하므로 모든 클라이언트가 동일한 적을 스폰

## 8. 신규 스프라이트

`sprite-generator.html`에 추가:

| Phaser 키 | 함수 | 크기 | 설명 |
|-----------|------|------|------|
| `enemy_idle` | `drawEnemy('idle')` | 32×32 | 적군 기본 (캐릭터와 유사하되 붉은 계열) |
| `enemy_walk` | `drawEnemy('walk')` | 32×32 | 적군 이동 |
| `particle_fire` | `drawFireParticle()` | 8×8 | 불꽃 파티클 텍스처 |

> 적군 스프라이트는 캐릭터(`drawCharacter`)와 동일한 구조로,
> 옷 색상만 붉은 계열(`#8b0000`, `#cc2222`)로 변경.
