# 설계 10 — 명령 큐 시스템

> **전제 조건**: 09 단계 완료 상태.
> 상호작용 시스템(벌목/채굴/낚시/건설), 이동(A*), 전투 Lock-on이 구현되어 있다고 가정한다.

---

## 1. 이번 단계 목표

1. Shift + 클릭으로 명령을 순차적으로 큐에 추가
2. 현재 명령 완료 시 다음 명령 자동 실행
3. 명령 큐 UI (화면 좌측 하단, 위→아래 순서로 표시)
4. 큐 실패/취소 처리

---

## 2. 큐 등록 방식

### 2-1. 기본 동작

| 입력 | 동작 |
|------|------|
| 일반 클릭 | 즉시 실행 (기존 동작, 큐 무시) |
| `Shift` + 클릭 | 현재 큐의 **맨 뒤에 명령 추가** |
| `Shift` + 클릭 (큐 비어있고 현재 동작 없음) | 즉시 실행 시작 + 큐 모드 진입 |

### 2-2. 큐에 추가 가능한 명령 종류

| 명령 | 대상 | 비고 |
|------|------|------|
| 이동 | 빈 타일 | 목적지까지 이동 |
| 벌목 | 나무 오브젝트 | 이동 후 벌목 |
| 채굴 | 암반 타일 | 이동 후 채굴 |
| 낚시 | 물 타일 | 이동 후 낚시 |
| 공격 | 동물 / 적군 | 이동 후 공격 (1회 또는 처치까지 — 추후 확정) |
| 건설 | 설치 위치 | 이동 후 건설 |
| 제작 | 작업대 | 이동 후 제작 1회 |
| 요리 | 조리대 | 이동 후 요리 1회 |

### 2-3. 큐에 추가할 수 없는 명령

- 인벤토리 열기 (`V`)
- 건설 패널 열기 (`B`)
- 수면 (침대 상호작용) — 수면 진입 시 큐 전체 일시중단

---

## 3. 명령 실행 흐름

```
[큐에 명령 존재]
        │
        ▼
큐 맨 앞 명령 꺼냄
        │
        ├── 대상이 상호작용 거리 내?
        │       YES → 즉시 동작 실행
        │       NO  → A*로 대상 방향 자동 이동 시작
        │               → 거리 내 도달 시 동작 실행
        │
        ▼
동작 완료 (프로그레스 바 완료 / 이동 완료)
        │
        ├── 큐에 다음 명령 있음 → 다음 명령 실행
        └── 큐 비어있음 → 큐 모드 종료
```

### 3-1. 명령 간 이동

- 각 명령 실행 전 A* 경로탐색으로 자동 이동
- 이동 중에도 큐 UI는 표시 유지
- 이동 경로는 다음 명령 대상을 기준으로 계산

---

## 4. 큐 취소 / 실패 처리

| 상황 | 처리 방식 |
|------|-----------|
| 일반 클릭 (Shift 없이) | **큐 전체 초기화** 후 클릭한 명령 즉시 실행 |
| `ESC` 키 | 큐 전체 초기화, 현재 동작 중단 |
| 방향키 입력 | 큐 전체 초기화, 수동 이동 모드로 전환 |
| 피격 (공격 받음) | 현재 명령 중단, 큐 유지 → 피격 후 자동 재개 |
| 광란 모드 돌입 | 큐 전체 초기화 (조작 불가) |
| 강제 수면 (피로 0) | 큐 일시 중단 → 수면 해제 후 자동 재개 |
| 대상 소멸 (나무 벌목됨, 동물 사망 등) | 해당 명령 스킵, 큐 UI에 ❌ 표시 후 0.5초 뒤 제거, 다음 명령으로 진행 |
| 재료 부족 (건설/제작) | 해당 명령 스킵, 큐 UI에 ⚠ 표시, 다음 명령으로 진행 |
| 인벤토리 가득 참 | 채집 명령 스킵, ⚠ 표시 |

---

## 5. 명령 큐 UI

### 5-1. 위치 및 레이아웃

```
화면 좌측 하단 고정
위에서 아래 방향으로 쌓임 (1번이 맨 위 = 다음 실행될 명령)

┌─────────────────────┐
│ ① 🪓 벌목  [나무]  │  ← 현재 실행 중 (강조 표시)
│ ② 🪓 벌목  [나무]  │
│ ③ ⛏ 채굴  [암반]  │
│ ④ 🏗 건설  [벽]    │
│ ⑤ 🎣 낚시  [물]    │
└─────────────────────┘
```

### 5-2. 카드 디자인

```
┌─────────────────────────┐
│ [아이콘] [명령명] [대상] │  높이: 28px
│ ▓▓▓▓▓░░░░░  (진행 바)  │  (실행 중인 명령만)
└─────────────────────────┘

상태별 스타일:
  실행 중: 흰 테두리, 배경 반투명 파랑
  대기 중: 회색 테두리, 배경 반투명 어두운 회색
  실패/스킵: 빨간 테두리, ❌ 아이콘, 0.5초 후 제거
  재료 부족: 노란 테두리, ⚠ 아이콘
```

### 5-3. 큐 카드 상호작용

| 동작 | 결과 |
|------|------|
| 카드 우클릭 | 해당 명령만 삭제 |
| 카드 드래그 | 순서 변경 |
| 큐 전체 지우기 버튼 (×) | 큐 초기화 |

---

## 6. 큐 추가 시 시각 피드백

Shift 누른 상태에서 대상 호버 시:
- 툴팁에 **"+ 큐에 추가"** 문구 표시
- 클릭 시 큐 UI에 해당 카드가 아래로 슬라이드 인 애니메이션으로 추가

---

## 7. 확정된 규칙

### A. 큐 최대 길이: **20개**
- 20개 초과 시 추가 불가
- 큐 UI 상단에 `17/20` 형식으로 현재 개수 표시
- 초과 시도 시 툴팁: "명령 큐가 가득 찼습니다 (20/20)"

### B. 공격 명령 범위: **대상 처치까지**
- Shift + 동물/적 클릭 → 해당 대상이 사망할 때까지 연속 공격
- 처치 완료 시 다음 큐 명령으로 자동 전환
- 대상이 도망가면 (사거리 초과) 자동 추격 후 재공격

### C. 피격 시 처리: **큐 전체 초기화**
- 피격 순간 현재 명령 중단 + 큐 전체 삭제
- **공격 명령 실행 중 피격도 동일하게 초기화** (예외 없음)
- 큐 UI 카드들이 빠르게 페이드 아웃되며 사라짐
- 이후 플레이어가 직접 다시 명령 입력

### D. 같은 대상 반복 추가: **별도 카드**
- 나무 3그루 Shift 클릭 → 벌목 카드 3개 각각 생성
- 같은 나무를 Shift 클릭 2회 → 동일 나무 벌목 카드 2개 (첫 번째 완료 후 두 번째도 시도 → 대상 소멸로 스킵)

### E. 맵 경계 넘는 명령: **불허**
- Shift 클릭 대상이 현재 맵 외부 타일인 경우 큐에 추가 불가
- 툴팁: "다른 맵의 대상은 큐에 추가할 수 없습니다"
- 현재 맵 안의 대상만 큐 등록 가능
